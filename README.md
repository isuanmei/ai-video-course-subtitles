# AI影像课程字幕 Skill

把一节剪好的课程，变成一条真正可以发布的成片。

这个 Codex skill 是为「AI影像实战课程」沉淀出来的字幕工作流。它不是简单地把转写文本糊到画面底部，而是把口播、片头、已有字幕、案例画面、专业名词和最终质检放进同一条流程里，让字幕像成片的一部分，而不是最后临时补上的一层字。

## 它解决什么

很多课程视频卡在最后一步：剪辑已经完成，内容也很完整，但字幕一上就容易出问题。

- 片头不该有字幕，却提前出现了口播文字
- 字幕和声音错开，观众会立刻觉得不专业
- 一行字幕太长，看不完，也压迫画面
- 已经有字幕的案例视频，又被重复叠了一层字幕
- AI 工具名被识别错：即梦、可灵、vidu、Seedance、GPT Image
- 专业词被识别错：首尾帧、图生视频、文生视频、一镜到底
- 修改一次错词后，重新生成又丢了修正

这个 skill 的目标就是把这些坑一次性收进流程里，让下一次上字幕更快、更稳、更像一个成熟课程产品。

## 核心原则

- 片头部分不要有字幕
- 字幕从正文口播声音开始，并与声音时间码保持一致
- 一行字幕通常是一句话，或一个清晰的语义单位
- 字幕不要太长，长句按自然停顿拆开
- 已经有字幕、歌词、案例视频字幕的地方不要重复上字幕
- 专名必须稳定：即梦、可灵、vidu、Seedance、GPT Image
- 每次烧录前后都要做文本扫描和抽帧检查

## 适合谁

这个 skill 适合正在做课程、训练营、录播课、实操课的人，尤其适合这种视频：

- 口播加屏幕录制
- 课程里穿插案例视频
- 画面中已经有课件文字或素材字幕
- 经常出现 AI 工具名和专业术语
- 希望字幕风格统一，而不是每一课都重新摸索

## 安装

把仓库 clone 到 Codex skills 目录：

```bash
git clone https://github.com/isuanmei/ai-video-course-subtitles.git \
  ~/.codex/skills/ai-video-course-subtitles
```

如果目录已存在，可以进入目录后更新：

```bash
cd ~/.codex/skills/ai-video-course-subtitles
git pull
```

## 调用方式

在 Codex 里直接说：

```text
用 $ai-video-course-subtitles 给成片文件夹里的 AI影像实战第六课无字幕版上字幕
```

也可以自然一点：

```text
给成片里的第六课无字幕版上字幕，按之前 AI影像实战课的字幕规范来
```

最稳的方式是显式带上 `$ai-video-course-subtitles`。

## 它会怎么做

1. 找到成片视频并读取视频参数
2. 建立独立字幕工作区
3. 提取音频并转写
4. 清洗字幕、校正专名、处理错词
5. 避开片头和已有字幕区间
6. 按一句话或自然停顿拆分字幕
7. 生成 SRT 和 ASS
8. 按统一样式烧录字幕
9. 抽帧检查关键时间点
10. 扫描常见错词并确认输出参数

## 字幕风格

默认风格参考 AI影像实战课第一课：

- 底部居中
- 白字
- 黑色描边
- 一屏一行
- 不加句尾标点
- 适合 4K 课程视频

## 已沉淀的课程词表

常见专名：

- 即梦
- 可灵
- vidu
- Seedance
- GPT Image

常见专业词：

- 首尾帧
- 图生视频
- 文生视频
- 一镜到底
- 音画同出

已记录的误识别：

- 吉梦、极梦、吉孟、寂寞 -> 即梦
- 克林、可林、可怜 -> 可灵
- 微杜、微度 -> vidu
- 手尾针、守卫针 -> 首尾帧
- 花瓣女孩 -> 滑板女孩

## 为什么值得用

好字幕不是装饰。它会影响一门课的完成度、节奏感和信任感。

当字幕准时出现、长度舒服、专名稳定、没有重复遮挡，观众会更容易进入内容，课程也会显得更可靠。这个 skill 的价值不是替你省下几分钟，而是把「最后交付前最容易翻车的细节」变成一套可以复用的标准。

如果你也在做知识产品、AI课程、实操教程，欢迎直接拿去用，也欢迎继续把新的错词和经验沉淀进去。
